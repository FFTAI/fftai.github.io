Clear Fault

Clear Fault

This example code demonstrates how to use the FSA (Fourier Smart Actuator) SDK to clear faults in FSA actuators. It is a part of the FSA SDK’s collection of example scripts.

Code Description

The clear_fault.py script does the following:

  1. Discovers FSA actuators on the network.

  2. Retrieves the error code for each detected actuator.

  3. Clears any faults present in the actuators, if any.

Dependencies

This script requires the FSA SDK and its dependencies to be installed. You can install the required dependencies using pip by running the following command:

python3 -m pip install -r requirements.txt

Usage

To run the clear_fault.py script, follow these steps:

  1. Ensure that you have set up your Python environment and installed the FSA SDK as mentioned in the Quickstart Guide.

  2. Open a terminal and navigate to the directory containing clear_fault.py.

  3. Run the script using the following command:

    python clear_fault.py

Common Issues

If you encounter any issues while running this script, consider the following:

  1. Ensure that you are using a compatible version of Python as recommended in the Quickstart Guide.

  2. Verify that your network is properly configured, and the FSA actuators are accessible.

  3. Check if the actuators have any existing faults that need to be cleared.

About FSA

The Fourier Smart Actuator (FSA) is a cutting-edge actuator technology developed for robotics and automation applications. The FSA SDK provides developers with the tools and resources to interact with FSA actuators and leverage their capabilities.

For more information about FSA and additional example scripts, refer to the FSA SDK Documentation.

Current Control Mode

Control Currrent Mode

Overview

FSAs support various control modes, each dictating a specific type of control applied to the actuator. These modes include:

  • Current Close-Loop Control

  • Velocity Control

  • Position Control

  • Trapezoidal Control

The script allows users to seamlessly switch between these control modes based on their application requirements.

Current Control

The set_current_control function is a key feature of the script, enabling users to command the FSA device to perform a current control operation. This operation involves setting a desired current before initiating the control process. Upon completion, the script provides a detailed response, including the current position, velocity, and current values. This functionality enhances precision and flexibility when interacting with FSAs in real-world applications.

Dependencies

  • Ensure that the fsa.fi_fsa module is properly imported.

  • The script requires Python 3.

  • Install dependencies using the following command:

    python3 -m pip install -r requirements.txt

Usage

  1. Specify the IP addresses of FSAs in the server_ip_list.

  2. Run the script to perform control operations on the FSAs.

Example Operations

The script demonstrates the following operations on each Actuator:

  1. Check State:

    • Obtain and print the state of each Actuator.

  2. Retrieve Configuration:

    • Fetch configuration details from each Actuator.

  3. Get Position, Velocity, and Current:

    • Retrieve and print Position, Velocity, and Current values from each Actuator.

  4. Set Current Control:

    • Set the current control current to 0.0 for each Actuator.

  5. Set Mode of Operation:

    • Set work at current control mode for each Actuator.

  6. Enable Motors:

    • Enable all the motors for each Actuator.

  7. Additional Control Operations:

    • Perform additional control operations (modify script accordingly).

  8. Disable Motors:

    • Disable the motors for each Actuator.

  9. Set Mode of Operation (None Control):

    • Set work at none control mode for each Actuator.

Running the Script

Run the script using the following command:

python3 control_current_mode.py

Velocity Control Mode

Velocity Control Mode

This example code, demo_control_velocity_mode.py, showcases the utilization of the FSA (Fourier Smart Actuator) SDK to control actuators in the velocity control mode. It is a part of the FSA SDK’s collection of example scripts.

Getting Started

Before running this example, ensure that the FSA SDK is installed correctly and configured. Refer to the FSA SDK Quickstart Guide for comprehensive instructions on setting up your Python environment and configuring the SDK.

Code Description

The demo_control_velocity_mode.py script performs the following operations:

  1. Discovers FSA actuators on the network.

  2. Retrieves the state, configuration, and position-velocity-current (PVC) information for each detected actuator.

  3. Sets the velocity control to 0.0 for all actuators.

  4. Enables all motors.

  5. Configures actuators to operate in the velocity control mode.

  6. Executes a sine wave motion for 10 seconds using velocity control.

  7. Disables all motors.

  8. Resets actuators to operate in the none control mode.

Dependencies

This script relies on the FSA SDK and its dependencies. You can install the required dependencies using pip by running the following command:

bash
python3 -m pip install -r requirements.txt

Usage

To run the demo_control_velocity_mode.py script, follow these steps:

  1. Ensure that you have set up your Python environment and installed the FSA SDK as mentioned in the Quickstart Guide.

  2. Open a terminal and navigate to the directory containing demo_control_velocity_mode.py.

  3. Run the script using the following command:

    bash
    python demo_control_velocity_mode.py

Common Issues

If you encounter any issues while running this script, consider the following:

  1. Verify that you are using a compatible version of Python as recommended in the Quickstart Guide.

  2. Ensure that your network is properly configured, and the FSA actuators are accessible.

  3. Check for any existing faults in the actuators that might affect their operation.

Module contents

FSA (Fourier Smart Actuator) SDK-Python Package

This package provides an SDK for interacting with FSA (Fourier Smart Actuator) actuator. It includes modules for various functionalities such as control, configuration, and demos.

Package Structure: - fi_fsa.py: Main module containing FSA class definitions and core functionality. - demo_* files: Demo scripts demonstrating the usage of FSA SDK features. - pyscurve: Submodule for trajectory planning using SCurve.

For detailed information about FSA and usage examples, refer to the documentation.
